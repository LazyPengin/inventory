# Postmortems

This folder contains **blameless postmortems** for incidents and outages.

## Philosophy
- No blame
- Focus on systems, not people
- Learn and improve continuously

## When to write a postmortem
Write a postmortem when:
- Users are impacted
- An SLO or SLA is violated
- A serious near-miss occurs
- A rollback is required

## What a postmortem should include
- What happened
- Customer impact
- Root cause
- What went well
- What went wrong
- Action items with owners
