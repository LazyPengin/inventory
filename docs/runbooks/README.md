# Runbooks

This folder contains **operational runbooks** for production systems.

## What belongs here
- How to deploy and rollback services
- Common incidents and how to resolve them
- Alarm definitions and response steps
- Debugging checklists
- Links to dashboards and logs

## When to add a runbook
Add a runbook when:
- A service is deployed to production
- A feature has operational impact
- On-call responsibility exists

## Runbook expectations
Each runbook should answer:
- What does this service/feature do?
- How do I know it is healthy?
- What are the common failure modes?
- How do I fix issues quickly?
- How do I rollback safely?
